# 修改应用图标方案

## 目标
将 `./icon.svg` 设为 AI Dictionary 应用的桌面图标与系统托盘图标，颜色为白色，背景透明。

## 现状
- 项目根目录存在 `./icon.svg`（Lucide languages 图标，stroke="currentColor"，fill="none"）。
- `src-tauri/icons/` 下为 Tauri 默认生成的多尺寸 PNG / ICO / ICNS / iOS 图标。
- `tauri.conf.json` 的 `bundle.icon` 指向 `icons/` 下的 PNG 文件。
- `src-tauri/src/lib.rs` 中托盘图标通过 `include_bytes!("../icons/tray_icon.png")` 加载。
- Tauri CLI (`bun run tauri`) 已安装，版本 2.11.0。
- 系统已安装 `rsvg-convert`，可用于 SVG → PNG 转换。

## 实施步骤

### 1. 准备白色 SVG 源文件
- 将 `./icon.svg` 复制到 `src-tauri/icons/icon.svg`。
- 将其中 `stroke="currentColor"` 修改为 `stroke="white"`，确保图标显示为白色。
- 原文件已设置 `fill="none"`，背景保持透明，无需额外修改。

### 2. 生成应用图标源 PNG
- 使用 `rsvg-convert` 将 `src-tauri/icons/icon.svg` 渲染为 1024×1024 的 `src-tauri/icons/icon.png`。
- 这是 Tauri `icon` 命令生成多平台图标所需的源文件。

### 3. 生成所有平台应用图标
- 运行 `bun run tauri icon src-tauri/icons/icon.png`。
- 该命令会自动生成并覆盖以下文件：
  - `icon.ico`（Windows 主图标）
  - `icon.icns`（macOS 主图标）
  - `ios/` 目录（iOS 应用图标）
  - `32x32.png`、`128x128.png`、`128x128@2x.png`
  - `Square107x107Logo.png` ~ `Square310x310Logo.png`
  - `StoreLogo.png`
- 生成后 `tauri.conf.json` 中的 `bundle.icon` 路径无需改动，仍指向 `icons/` 下的 PNG 文件。

### 4. 生成托盘图标
- 使用 `rsvg-convert` 将白色 SVG 渲染为 `src-tauri/icons/tray_icon.png`（32×32 或 64×64）。
- 同时更新 `src-tauri/icons/tray_icon.svg` 为白色版本，保持文件一致性。
- `lib.rs` 中已引用 `tray_icon.png`，路径无需修改。

### 5. 清理旧图标
- 删除 `src-tauri/icons/app_icon.svg`（旧的默认图标）。

## 涉及文件
| 文件 | 操作 |
|------|------|
| `./icon.svg` | 读取（源文件） |
| `src-tauri/icons/icon.svg` | 写入（白色版本） |
| `src-tauri/icons/icon.png` | 覆盖（高清源 PNG） |
| `src-tauri/icons/tray_icon.png` | 覆盖（白色托盘图标） |
| `src-tauri/icons/tray_icon.svg` | 覆盖（白色版本） |
| `src-tauri/icons/app_icon.svg` | 删除（旧默认图标） |
| `src-tauri/icons/*.{ico,icns,png}` | 由 Tauri CLI 自动生成/覆盖 |
| `src-tauri/tauri.conf.json` | 无需修改 |
| `src-tauri/src/lib.rs` | 无需修改 |

## 验证
- 运行 `bun run dev:pc` 检查应用窗口图标是否正确显示。
- 检查系统托盘区域是否显示白色透明背景的图标。
- 检查生成的 `icons/` 目录下文件时间戳是否已更新。

## 风险与注意事项
- `bun run tauri icon` 会覆盖 `icons/` 目录下同名文件，请确保没有自定义图标需要保留。
- 托盘图标在不同操作系统上的显示效果可能有差异（如 macOS 会自动反色），白色在深色任务栏上显示效果最佳。
